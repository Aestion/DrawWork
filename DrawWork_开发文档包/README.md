## 项目概况

- **项目名称**: DrawWork
- **用途**: 公司内部在线协作白板
- **部署方式**: 内网服务器部署
- **开发工期**: 11天
- **技术栈**: React + Node.js + PostgreSQL + Docker

## 核心功能

1. **画板管理** - 创建多个画板，自定义命名
2. **多画布** - 每个画板可包含多个画布，可排序
3. **分享协作** - 以画板为单位分享，支持编辑/查看/评论权限
4. **富媒体** - 支持图片、GIF（自动循环）、视频（自动循环）、音频（点击播放）
5. **思维导图** - 创建思维导图，支持 Markdown 导入导出
6. **泳道图** - 水平/垂直泳道，适合流程分析
7. **看板** - 多列任务管理，拖拽移动卡片
8. **评论** - 画布任意位置添加评论，支持回复和 @ 提及
9. **投票** - 对画布元素发起投票，实时计票
10. **计时器** - 主持人控制倒计时，全员可见
11. **实时协作** - 多人同时编辑，光标同步

## 快速开始

```bash
# 1. 准备服务器（Ubuntu 22.04，4核8G+）
# 2. 安装 Docker

# 3. 克隆项目
git clone https://github.com/your-company/drawwork.git
cd drawwork

# 4. 配置环境变量
cp config/.env.example config/.env
vim config/.env  # 修改密码等配置

# 5. 一键部署
./scripts/deploy.sh

# 6. 访问
# http://服务器IP
# 默认账号: admin / admin123
```

## 系统架构

```
用户浏览器
    ↓
Nginx（反向代理 + 静态文件）
    ↓
├── API 服务（Node.js + Express）←→ PostgreSQL（数据库）
├── Yjs 服务（WebSocket）←→ Redis（缓存）
└── Minio（文件存储）
```

## 稳定性优先建议

如果不以工期为主要约束，后续开发以 [08_稳定性优先开发方案.md](./08_稳定性优先开发方案.md) 为准，优先处理权限边界、协作持久化、内部服务暴露、文件访问控制和备份恢复验证。

## 注意事项

1. **部署前必做**:
   - 修改默认密码（数据库、JWT、Minio）
   - 配置内网域名或IP
   - 配置防火墙规则

2. **生产环境建议**:
   - 启用自动备份
   - 配置监控告警
   - 定期更新系统

3. **安全提醒**:
   - 内网部署，勿暴露公网
   - 定期更换管理员密码
   - 及时更新依赖包
