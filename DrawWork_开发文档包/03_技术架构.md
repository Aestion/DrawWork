## 二、组件说明

### 2.1 Nginx（反向代理）

**职责**：
- 提供前端静态文件服务
- API 请求转发到 Node.js 后端
- WebSocket 连接转发到 Yjs 服务
- 负载均衡（多实例时）
- SSL/TLS 终端（如需 HTTPS）

**关键配置**：
```nginx
# 前端静态文件
location / {
    root /usr/share/nginx/html;
    try_files $uri $uri/ /index.html;
}

# API 代理
location /api/ {
    proxy_pass http://api:3000/;
    proxy_http_version 1.1;
}

# WebSocket 代理
location /ws {
    proxy_pass http://yjs:3001;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

### 2.2 API 服务（Node.js + Express）

**职责**：
- 用户认证（JWT）
- 画板/画布 CRUD
- 分享管理
- 文件上传接口
- 系统管理接口

**技术栈**：
- Node.js 20 LTS
- Express 4.x
- Sequelize ORM
- bcrypt（密码加密）
- jsonwebtoken（JWT）
- multer（文件上传）

### 2.3 Yjs 服务（WebSocket）

**职责**：
- 实时协作同步（Excalidraw 文档 CRDT）
- Yjs 文档管理与持久化
- 权限验证（连接时校验 JWT + 画板权限）
- 评论实时推送
- 投票状态同步
- 计时器同步（主持人控制，全员可见）
- 通知推送（新评论、@提及、投票结果）

**技术栈**：
- Node.js 20 LTS
- y-websocket
- yjs
- ws (WebSocket 库)

### 2.4 PostgreSQL（数据库）

**职责**：
- 用户数据
- 画板/画布元数据
- 分享信息
- Yjs 文档快照
- 操作日志

**版本**：PostgreSQL 15

### 2.5 Redis（缓存）

**职责**：
- 用户会话缓存
- WebSocket 连接状态
- 在线用户列表
- 限流计数
- Pub/Sub 消息总线（评论/投票/通知事件传递）

**版本**：Redis 7

### 2.6 Minio（文件存储）

**职责**：
- 图片文件存储
- 视频文件存储
- 音频文件存储
- 画板封面存储

**版本**：Minio Latest

### 3.4 评论同步流程

```
用户 A 添加评论              用户 B 界面
    │                            ▲
    │                            │
    ▼                            │
┌───────────┐             ┌───────────┐
│ API 服务  │             │ WebSocket │
│ POST /comments           │ 推送事件  │
└─────┬─────┘             └─────┬─────┘
      │                         ▲
      │ 写入数据库               │ 推送
      ▼                         │
┌───────────┐    WebSocket   ┌───────────┐
│ PostgreSQL │               │  Yjs 服务  │
│ comments 表│◀──────────────│ 广播给同房间│
└───────────┘               └───────────┘
```

**流程说明**：
1. 用户 A 在画布上添加评论 → 调用 API 写入数据库
2. API 写入成功后，通过 Redis pub/sub 发布事件到 `canvas:{canvasId}` 频道
3. Yjs 服务订阅 Redis 频道，收到事件后通过 WebSocket 广播给同一画布的所有在线用户
4. 用户 B 界面实时显示新评论锚点

```
主持人操作                   所有用户界面
    │                            ▲

    ▼                            │
┌───────────┐    WebSocket    ┌───────────┐

    ┌──────────────────────────────┘
    │ 最后1分钟
    ▼
┌───────────┐
│ 闪烁提醒  │
└─────┬─────┘
      │
      │ 时间到

┌───────────┐

```

**说明**：
- 计时器状态存储在 Yjs 服务内存中（不持久化到数据库）
- 主持人启动/暂停/重置操作通过 WebSocket 广播
- 各客户端本地执行倒计时，保证显示一致性
- 时间到达时由 Yjs 服务广播结束事件

### 3.7 通知推送流程

```
触发事件（评论@提及/分享邀请/投票结束）
    │
    ▼
┌───────────┐
│  API 服务  │
│ 写入通知表 │
└─────┬─────┘
      │
      │ Redis pub/sub 发布到 user:{userId} 频道
      ▼
┌───────────┐    WebSocket    ┌───────────┐
│  Yjs 服务  │═══════════════▶│ 目标用户   │
│ 推送通知   │                │ 站内通知弹窗│
└───────────┘                └───────────┘
```

**说明**：
- 通知通过 Redis pub/sub 从 API 服务传递到 Yjs 服务
- Yjs 服务维护用户 WebSocket 连接，实时推送给在线用户
- 离线用户上线后通过轮询 API 获取未读通知

## 四、安全设计

### 4.1 认证安全

- **JWT Token**：HS256 签名，包含用户ID、权限、过期时间
- **Token 有效期**：Access Token 24小时，Refresh Token 7天
- **密码加密**：bcrypt，cost factor 12
- **会话管理**：Redis 存储，支持强制下线

### 4.2 权限控制

| 层级 | 控制方式 |
|------|----------|
| API 层 | JWT 验证 + 权限检查中间件 |
| WebSocket | 连接时验证 Token + 画板权限 |
| 数据库 | 外键约束 + 级联删除 |

### 4.3 数据安全

- **传输加密**：内网可选 HTTP，外网建议 HTTPS
- **存储加密**：数据库敏感字段加密（密码）
- **备份加密**：备份文件可配置加密
- **审计日志**：记录关键操作

```
                    ┌──────────────┐
                    │    Nginx     │
                    │   负载均衡   │
                    └──────┬───────┘
                           │
           ┌───────────────┼───────────────┐

      ┌─────────┐    ┌─────────┐    ┌─────────┐
      │ Yjs 1   │    │ Yjs 2   │    │ Yjs 3   │

                    │    Redis     │
                    │  Yjs 适配器  │
                    └──────────────┘
```

**实现方式**：
- 多个 Yjs 服务实例
- Redis 作为 Yjs 适配器，实现跨实例同步
- Nginx ip_hash 保持会话

### 5.2 高可用

| 组件 | 高可用方案 |
|------|----------|
| Nginx | 主备部署或 Keepalived |
| API | 多实例 + 负载均衡 |
| Yjs | 多实例 + Redis 同步 |
| PostgreSQL | 主从复制 |
| Redis | Sentinel 模式 |
| Minio | 分布式部署 |
