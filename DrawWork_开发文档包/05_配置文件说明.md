# DrawWork 配置文件说明

## 配置文件清单

| 文件 | 用途 | 位置 |
|------|------|------|
| docker-compose.yml | Docker 服务编排 | config/ |
| nginx.conf | Nginx 反向代理配置 | config/ |
| .env | 环境变量 | config/ |
| backup.sh | 自动备份脚本 | scripts/ |
| update.sh | 系统更新脚本 | scripts/ |

## 一、docker-compose.yml

### 完整配置

```yaml
version: '3.8'

services:
  # Nginx 反向代理
  nginx:
    image: nginx:alpine
    container_name: drawwork-nginx
    ports:
      - "80:80"
      - "443:443"  # 如需 HTTPS
    volumes:
      - ./config/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./frontend/dist:/usr/share/nginx/html:ro
      - ./data/uploads:/var/www/uploads:ro
      - ./logs/nginx:/var/log/nginx
    depends_on:
      - api
      - yjs
    networks:
      - drawwork-network
    restart: always

  # API 后端服务
  api:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: drawwork-api
    environment:
      - NODE_ENV=production
      - PORT=3000
      - DATABASE_URL=postgres://${DB_USER:-postgres}:${DB_PASSWORD:-drawwork123}@postgres:5432/${DB_NAME:-drawwork}
      - REDIS_URL=redis://redis:6379
      - JWT_SECRET=${JWT_SECRET}
      - JWT_EXPIRES_IN=24h
      - MINIO_ENDPOINT=minio
      - MINIO_PORT=9000
      - MINIO_ACCESS_KEY=${MINIO_ACCESS_KEY:-minioadmin}
      - MINIO_SECRET_KEY=${MINIO_SECRET_KEY:-minioadmin123}
      - MINIO_BUCKET=drawings
      - UPLOAD_MAX_SIZE=104857600  # 100MB
    volumes:
      - ./logs/api:/app/logs
      - ./data/uploads:/app/uploads
    depends_on:
      - postgres
      - redis
      - minio
    networks:
      - drawwork-network
    restart: always
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

  # Yjs WebSocket 服务
  yjs:
    build:
      context: ./yjs-server
      dockerfile: Dockerfile
    container_name: drawwork-yjs
    environment:
      - NODE_ENV=production
      - PORT=3001
      - DATABASE_URL=postgres://${DB_USER:-postgres}:${DB_PASSWORD:-drawwork123}@postgres:5432/${DB_NAME:-drawwork}
      - REDIS_URL=redis://redis:6379
      - API_URL=http://api:3000
      - SAVE_INTERVAL=30000  # 30秒保存一次
    ports:
      - "3001:3001"
    depends_on:
      - redis
      - api
    networks:
      - drawwork-network
    restart: always

  # PostgreSQL 数据库
  postgres:
    image: postgres:15-alpine
    container_name: drawwork-postgres
    environment:
      - POSTGRES_USER=${DB_USER:-postgres}
      - POSTGRES_PASSWORD=${DB_PASSWORD:-drawwork123}
      - POSTGRES_DB=${DB_NAME:-drawwork}
      - PGDATA=/var/lib/postgresql/data/pgdata
    volumes:
      - ./data/postgres:/var/lib/postgresql/data
      - ./backups:/backups
      - ./config/init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    networks:
      - drawwork-network
    restart: always
    command: >
      postgres
      -c shared_buffers=256MB
      -c effective_cache_size=768MB
      -c maintenance_work_mem=64MB
      -c work_mem=4MB

  # Redis 缓存
  redis:
    image: redis:7-alpine
    container_name: drawwork-redis
    volumes:
      - ./data/redis:/data
      - ./config/redis.conf:/usr/local/etc/redis/redis.conf:ro
    networks:
      - drawwork-network
    restart: always
    command: redis-server --appendonly yes --maxmemory 256mb --maxmemory-policy allkeys-lru

  # Minio 文件存储
  minio:
    image: minio/minio:latest
    container_name: drawwork-minio
    command: server /data --console-address ":9001"
    environment:
      - MINIO_ROOT_USER=${MINIO_ACCESS_KEY:-minioadmin}
      - MINIO_ROOT_PASSWORD=${MINIO_SECRET_KEY:-minioadmin123}
    volumes:
      - ./data/minio:/data
    networks:
      - drawwork-network
    restart: always

  # 数据库管理工具（可选）
  adminer:
    image: adminer
    container_name: drawwork-adminer
    ports:
      - "8080:8080"
    depends_on:
      - postgres
    networks:
      - drawwork-network
    restart: always
    profiles: ["admin"]  # 默认不启动，使用 docker-compose --profile admin up -d

networks:
  drawwork-network:
    driver: bridge
```

### 关键配置说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `services.nginx.ports` | 对外暴露的端口 | 80, 443 |
| `services.api.environment.DATABASE_URL` | 数据库连接字符串 | - |
| `services.api.environment.JWT_SECRET` | JWT 密钥（必须修改） | - |
| `services.postgres.environment.POSTGRES_PASSWORD` | 数据库密码 | drawwork123 |
| `services.minio.environment.MINIO_ROOT_PASSWORD` | Minio 管理员密码 | minioadmin123 |
| `services.yjs.environment.SAVE_INTERVAL` | Yjs 自动保存间隔（毫秒） | 30000 |

## 二、nginx.conf

### 完整配置

```nginx
user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
    use epoll;
    multi_accept on;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    # 日志格式
    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';

    access_log /var/log/nginx/access.log main;

    # 性能优化
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;

    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_proxied expired no-cache no-store private auth;
    gzip_types
        text/plain
        text/css
        text/xml
        text/javascript
        application/javascript
        application/xml+rss
        application/json;

    # 上游服务
    upstream api_backend {
        server api:3000;
        # 多实例时：
        # server api2:3000;
        # server api3:3000;
    }

    upstream yjs_backend {
        ip_hash;  # WebSocket 会话保持
        server yjs:3001;
        # 多实例时：
        # server yjs2:3001;
        # server yjs3:3001;
    }

    server {
        listen 80;
        server_name _;
        
        # 文件上传大小限制
        client_max_body_size 100M;
        client_body_buffer_size 16k;

        # 安全响应头
        add_header X-Frame-Options "SAMEORIGIN" always;
        add_header X-Content-Type-Options "nosniff" always;
        add_header X-XSS-Protection "1; mode=block" always;
        add_header Referrer-Policy "strict-origin-when-cross-origin" always;

        # 前端静态文件
        location / {
            root /usr/share/nginx/html;
            index index.html;
            try_files $uri $uri/ /index.html;

            # 静态资源缓存
            location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
                expires 1y;
                add_header Cache-Control "public, immutable";
                access_log off;
            }

            # HTML 不缓存
            location ~* \.html$ {
                expires -1;
                add_header Cache-Control "no-store";
            }
        }

        # API 代理
        location /api/ {
            proxy_pass http://api_backend;
            proxy_http_version 1.1;
            
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            
            proxy_connect_timeout 60s;
            proxy_send_timeout 60s;
            proxy_read_timeout 60s;
            
            # 错误处理
            proxy_intercept_errors on;
            error_page 502 503 504 /50x.html;
        }

        # WebSocket 代理
        location /ws {
            proxy_pass http://yjs_backend;
            proxy_http_version 1.1;
            
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            
            # WebSocket 长连接
            proxy_connect_timeout 7d;
            proxy_send_timeout 7d;
            proxy_read_timeout 7d;
            
            # 缓冲设置
            proxy_buffering off;
        }

        # 文件存储访问
        location /uploads/ {
            alias /var/www/uploads/;
            expires 1y;
            add_header Cache-Control "public";
            access_log off;
        }

        # 健康检查
        location /health {
            access_log off;
            return 200 "healthy\n";
            add_header Content-Type text/plain;
        }

        # 错误页面
        location = /50x.html {
            root /usr/share/nginx/html;
            internal;
        }

        # 禁止访问敏感文件
        location ~ /\. {
            deny all;
            access_log off;
            log_not_found off;
        }

        location ~* ^/(README|CHANGELOG|LICENSE|composer\.(json|lock)) {
            deny all;
            access_log off;
            log_not_found off;
        }
    }

    # HTTPS 配置（如需）
    # server {
    #     listen 443 ssl http2;
    #     server_name drawwork.company.local;
    #
    #     ssl_certificate /etc/nginx/ssl/cert.pem;
    #     ssl_certificate_key /etc/nginx/ssl/key.pem;
    #
    #     # 其他配置同上...
    # }
}
```

### 关键配置说明

| 配置项 | 说明 | 建议值 |
|--------|------|--------|
| `client_max_body_size` | 最大上传文件大小 | 100M |
| `upstream yjs_backend.ip_hash` | WebSocket 会话保持 | 必须开启 |
| `proxy_read_timeout` (WebSocket) | WebSocket 连接超时 | 7d (长连接) |
| `location /uploads/` | 文件存储访问路径 | 映射到 Minio 数据目录 |

## 四、backup.sh（备份脚本）

```bash
#!/bin/bash
# ============================================
# DrawWork 自动备份脚本
# ============================================

set -e

# 配置
BACKUP_DIR="${BACKUP_DIR:-/opt/drawwork/backups}"
RETENTION_DAYS="${BACKUP_RETENTION_DAYS:-30}"
DATE=$(date +%Y%m%d_%H%M%S)
HOSTNAME=$(hostname)

# 确保备份目录存在
mkdir -p "$BACKUP_DIR"

echo "[$(date)] 开始备份..."

# 备份数据库
echo "[$(date)] 备份数据库..."
docker exec drawwork-postgres pg_dump -U postgres drawwork | gzip > "$BACKUP_DIR/db_${DATE}.sql.gz"

# 备份文件存储
echo "[$(date)] 备份文件..."
tar -czf "$BACKUP_DIR/uploads_${DATE}.tar.gz" -C /opt/drawwork/data uploads

# 备份配置文件
echo "[$(date)] 备份配置..."
tar -czf "$BACKUP_DIR/config_${DATE}.tar.gz" -C /opt/drawwork config

# 创建备份信息文件
cat > "$BACKUP_DIR/info_${DATE}.txt" << EOF
备份时间: $(date)
服务器: $HOSTNAME
数据库: db_${DATE}.sql.gz
文件: uploads_${DATE}.tar.gz
配置: config_${DATE}.tar.gz
EOF

echo "[$(date)] 备份完成:"
ls -lh "$BACKUP_DIR"/*_${DATE}.*

# 清理旧备份
echo "[$(date)] 清理 ${RETENTION_DAYS} 天前的备份..."
find "$BACKUP_DIR" -name "*.gz" -mtime +$RETENTION_DAYS -delete
find "$BACKUP_DIR" -name "*.txt" -mtime +$RETENTION_DAYS -delete

# 计算备份大小
BACKUP_SIZE=$(du -sh "$BACKUP_DIR" | cut -f1)
echo "[$(date)] 当前备份目录大小: $BACKUP_SIZE"

# 发送通知（如果配置了邮件）
if [ -n "$SMTP_HOST" ]; then
    echo "备份完成: $DATE, 大小: $BACKUP_SIZE" | mail -s "DrawWork 备份通知" admin@company.local
fi

echo "[$(date)] 备份流程结束"
```

### 使用方法

```bash
# 手动执行备份
./scripts/backup.sh

# 添加到定时任务（每天凌晨2点）
crontab -e
# 添加: 0 2 * * * /opt/drawwork/scripts/backup.sh >> /opt/drawwork/logs/backup.log 2>&1
```

## 五、update.sh（更新脚本）

```bash
#!/bin/bash
# ============================================
# DrawWork 更新脚本
# ============================================

set -e

PROJECT_DIR="${PROJECT_DIR:-/opt/drawwork}"
BACKUP_DIR="${BACKUP_DIR:-/opt/drawwork/backups}"
DATE=$(date +%Y%m%d_%H%M%S)

cd "$PROJECT_DIR"

echo "🚀 DrawWork 更新脚本"
echo "===================="

# 检查是否在项目目录
if [ ! -f "docker-compose.yml" ]; then
    echo "❌ 错误: 未找到 docker-compose.yml，请确保在项目根目录"
    exit 1
fi

# 1. 备份当前数据
echo "📦 步骤 1/6: 备份当前数据..."
if [ -f "scripts/backup.sh" ]; then
    ./scripts/backup.sh
else
    echo "⚠️ 警告: 未找到备份脚本，跳过备份"
fi

# 2. 拉取最新代码
echo "📥 步骤 2/6: 拉取最新代码..."
git pull origin main || echo "⚠️ 警告: 拉取代码失败，使用本地代码继续"

# 3. 拉取最新镜像
echo "🐳 步骤 3/6: 拉取最新 Docker 镜像..."
docker-compose pull

# 4. 停止服务
echo "🛑 步骤 4/6: 停止当前服务..."
docker-compose down

# 5. 运行数据库迁移
echo "🔄 步骤 5/6: 运行数据库迁移..."
docker-compose up -d postgres
sleep 5  # 等待数据库启动

# 检查是否有新迁移
if [ -f "backend/package.json" ]; then
    docker-compose run --rm api npm run migrate || echo "⚠️ 警告: 数据库迁移可能失败"
fi

# 6. 启动服务
echo "✅ 步骤 6/6: 启动服务..."
docker-compose up -d

# 等待服务启动
sleep 10

# 健康检查
echo "🏥 检查服务状态..."
if curl -f http://localhost/health > /dev/null 2>&1; then
    echo "✅ 服务运行正常"
else
    echo "❌ 警告: 服务可能未正常启动，请检查日志"
    echo "查看日志: docker-compose logs"
fi

echo ""
echo "===================="
echo "✅ 更新完成！"
echo "访问地址: http://localhost"
echo "备份位置: $BACKUP_DIR"
echo "===================="
```

### 使用方法

```bash
# 执行更新
./scripts/update.sh

# 查看更新后的日志
docker-compose logs -f
```
