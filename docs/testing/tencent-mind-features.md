# Tencent Mind 功能测试套件

> 腾讯思维导图（TencentMindEditor）完整功能测试用例
> 覆盖：节点 CRUD、拖拽、外框、标记、关联线、概要、富文本、媒体、保存/加载、格式转换

---

## 0. 测试环境

```bash
# 1. 启动后端
cd backend && npm start          # port 3001

# 2. 启动前端
cd frontend && npm run dev       # port 5173

# 3. 打开浏览器
# 访问 http://localhost:5173 并进入腾讯思维导图画布
```

---

## 1. 节点 CRUD

### TC-NODE-001: 创建子节点
**目标**: 验证"添加子节点"按钮在当前选中节点下创建子节点
**前置条件**: 画布已加载，存在至少一个节点
**步骤**:
1. 点击一个已有节点使其激活
2. 点击工具栏「添加子节点」按钮
**预期结果**:
- 在激活节点下创建一个新的子节点
- 新节点的标题为可编辑状态（空白或默认文本）
- 节点树结构更新

### TC-NODE-002: 创建同级节点
**目标**: 验证「添加同级」按钮在激活节点旁创建同级节点
**前置条件**: 画布已加载，存在至少一个节点
**步骤**:
1. 点击一个非根节点使其激活
2. 点击工具栏「添加同级」按钮
**预期结果**:
- 在激活节点同一层级创建一个同级节点
- 布局自适应调整

### TC-NODE-003: 删除节点
**目标**: 验证通过键盘删除节点
**前置条件**: 存在可删除的子节点
**步骤**:
1. 点击一个子节点激活
2. 按 Delete 或 Backspace 键
**预期结果**: 节点从画布中移除，子节点同时被删除

### TC-NODE-004: 重命名节点
**目标**: 验证双击节点可编辑文本
**前置条件**: 画布已加载
**步骤**:
1. 双击一个节点
2. 输入新文本
3. 按 Enter 或点击其他区域确认
**预期结果**: 节点标题更新为新文本

### TC-NODE-005: 多级嵌套
**目标**: 验证多层子节点嵌套创建
**前置条件**: 画布已加载
**步骤**:
1. 创建根节点的子节点 A
2. 选中 A，创建 A 的子节点 B
3. 选中 B，创建 B 的子节点 C
**预期结果**: 形成三级嵌套：根 → A → B → C，所有节点正常显示和交互

## 2. 拖拽

### TC-DRAG-001: 同级重新排序
**目标**: 验证拖拽节点可改变同级顺序
**前置条件**: 至少有两个同级节点
**步骤**:
1. 点击并拖动一个节点到同一层级不同位置
2. 释放鼠标
**预期结果**: 节点移动到新位置，同级节点重新排列

### TC-DRAG-002: 跨分支移动
**目标**: 验证节点可从一条分支拖到另一分支
**前置条件**: 存在至少两个不同分支的节点
**步骤**:
1. 选择一个子节点 A
2. 将 A 拖到另一个分支的节点 B 下方
3. 释放鼠标
**预期结果**: 节点 A 变为 B 的同级节点（同一父级下）

### TC-DRAG-003: 跨侧拖拽（一级节点左右侧移动）
**目标**: 验证一级节点可在左右两侧之间拖拽
**前置条件**: 存在至少两个一级节点分别位于左右侧
**步骤**:
1. 将一个一级节点从左侧拖到右侧区域
2. 释放鼠标
**预期结果**: 节点移动到右侧，右侧节点数量 +1，左侧 -1

### TC-DRAG-004: 取消拖拽
**目标**: 验证拖拽过程中可按 Esc 取消
**前置条件**: 存在至少一个节点
**步骤**:
1. 开始拖动一个节点
2. 在释放前按 Esc 键
**预期结果**: 节点回到原始位置，布局不变

## 3. 外框（Boundaries）

### TC-BOUND-001: 为单个节点添加外框
**目标**: 验证选中单个节点后点击「添加外框」可创建外框
**前置条件**: 存在至少一个子节点
**步骤**:
1. 激活一个节点（非根节点）
2. 点击工具栏「添加外框」按钮
**预期结果**:
- 该节点被外框包围（蓝色圆角边框 + 浅蓝背景）
- 外框样式: 蓝色描边 `#0984e3`，背景 `rgba(9,132,227,0.05)`

### TC-BOUND-002: 为多个节点添加外框
**目标**: 验证选中多个同级节点后添加外框覆盖所有选中节点
**前置条件**: 存在至少 2 个同级子节点
**步骤**:
1. 按住 Ctrl 点击多个同级节点
2. 点击「添加外框」
**预期结果**: 外框包含所有选中节点

### TC-BOUND-003: 删除外框
**目标**: 验证「删除外框」按钮可移除选中外框
**前置条件**: 存在至少一个外框
**步骤**:
1. 点击外框使其激活（选中状态）
2. 点击工具栏「删除外框」按钮
**预期结果**: 外框被移除，内部节点保持

### TC-BOUND-004: 外框保存后加载恢复
**目标**: 验证添加外框并保存后重新加载仍保留外框
**前置条件**: 画布有一个或多个外框
**步骤**:
1. 添加外框
2. 点击「保存」或等待自动保存
3. 刷新页面 / 重新进入画布
**预期结果**: 外框在重新加载后仍然显示

## 10. 格式转换（Conversion Round-Trip）

### TC-CONV-001: tencentToSimpleMindMap 完整转换
**目标**: 验证 `DEFAULT_TENCENT_MIND` 完整转换为 simple-mind-map 格式
**前置条件**: 导入 tencent-mind-utils
**步骤**:
1. 调用 `tencentToSimpleMindMap(DEFAULT_TENCENT_MIND)`
**预期结果**:
- 返回的根节点 `data.text` === "EE2"
- 每个节点包含 `data._tencentMeta.id`
- 存在标记的节点有 `_tencentMeta.markers`
- 存在外框的节点有 `_tencentMeta.boundaries`
- 关联线数据不丢失

### TC-CONV-002: simpleMindMapToTencent 完整转换
**目标**: 验证 simple-mind-map 数据正确转回 Tencent 格式
**前置条件**: 通过 tencentToSimpleMindMap 得到 smmData
**步骤**:
1. 调用 `simpleMindMapToTencent(smmData, origTencentData)`
**预期结果**:
- 返回 Tencent 格式包含 `rootTopic`
- `rootTopic.title.children[0].children[0].text` 包含标题文本
- 关联线保留在 `relationships` 数组中
- 主题/结构信息保留

### TC-CONV-003: 往返转换一致性（Identity）
**目标**: 验证 Tencent → simple-mind-map → Tencent 往返后核心数据不变
**前置条件**: 使用 DEFAULT_TENCENT_MIND 数据
**步骤**:
1. `const smm = tencentToSimpleMindMap(DEFAULT_TENCENT_MIND)`
2. `const back = simpleMindMapToTencent(smm, DEFAULT_TENCENT_MIND)`
**预期结果**:
- `back.rootTopic.title` 与原始 title 结构相似（至少文本内容一致）
- 所有原始节点的 id 保留在 `rootTopic` 树中
- 主要子节点数量一致
