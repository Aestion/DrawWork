# DrawWork 运维手册

## 一、日常运维命令

### 1.1 查看服务状态

```bash
# 进入项目目录
cd /opt/drawwork

# 查看所有服务状态
docker-compose ps

# 预期输出：
# NAME                IMAGE                      STATUS
drawwork-nginx      nginx:alpine               Up 3 days
drawwork-api        drawwork/api:latest        Up 3 days
drawwork-yjs        drawwork/yjs:latest        Up 3 days
drawwork-postgres   postgres:15-alpine         Up 3 days
drawwork-redis      redis:7-alpine             Up 3 days
drawwork-minio      minio/minio:latest         Up 3 days
```

### 1.2 查看日志

```bash
# 查看所有服务日志
docker-compose logs

# 查看最后 100 行日志
docker-compose logs --tail=100

# 实时查看日志（类似 tail -f）
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f api
docker-compose logs -f yjs
docker-compose logs -f postgres

# 查看 nginx 访问日志
docker exec drawwork-nginx tail -f /var/log/nginx/access.log

# 查看 nginx 错误日志
docker exec drawwork-nginx tail -f /var/log/nginx/error.log
```

### 1.3 重启服务

```bash
# 重启所有服务
docker-compose restart

# 重启特定服务
docker-compose restart api
docker-compose restart yjs

# 重新构建并重启（代码更新后）
docker-compose up -d --build api
```

### 1.4 停止和启动

```bash
# 停止所有服务
docker-compose stop

# 停止并删除容器
docker-compose down

# 停止并删除容器和卷（危险！会删除数据）
docker-compose down -v

# 启动服务
docker-compose up -d

# 完全重置（停止、删除、重新创建）
docker-compose down
docker-compose up -d
```

## 二、用户管理

### 2.1 创建管理员账号

```bash
# 进入数据库
docker exec -it drawwork-postgres psql -U postgres -d drawwork

# 创建用户（密码使用 bcrypt 加密）
INSERT INTO users (id, username, email, password_hash, is_admin, is_active)
VALUES (
  gen_random_uuid(),
  'newadmin',
  'newadmin@company.local',
  '$2b$12$LQv3c1yqBWVHxkd0LHAkCOYz6TtxMQJqhN8/LewKyNiAYMyzJ/I2K',  -- 密码: admin123
  TRUE,
  TRUE
);

# 创建资料
INSERT INTO profiles (id, username)
VALUES (
  (SELECT id FROM users WHERE username = 'newadmin'),
  'newadmin'
);

# 退出
\q
```

### 2.2 重置用户密码

```bash
# 进入数据库
docker exec -it drawwork-postgres psql -U postgres -d drawwork

# 更新密码（bcrypt hash）
UPDATE users 
SET password_hash = '$2b$12$LQv3c1yqBWVHxkd0LHAkCOYz6TtxMQJqhN8/LewKyNiAYMyzJ/I2K'
WHERE username = 'username';

# 退出
\q
```

### 2.3 禁用用户

```bash
docker exec -it drawwork-postgres psql -U postgres -d drawwork

UPDATE users SET is_active = FALSE WHERE username = 'username';

\q
```

### 2.4 查看用户列表

```bash
docker exec -it drawwork-postgres psql -U postgres -d drawwork -c \
  "SELECT username, email, is_admin, is_active, created_at FROM users ORDER BY created_at DESC;"
```

## 三、备份与恢复

### 3.1 手动备份

```bash
# 执行备份脚本
cd /opt/drawwork
./scripts/backup.sh

# 备份文件位置
ls -lh backups/

# 预期输出：
# db_20260425_020000.sql.gz      # 数据库备份
# uploads_20260425_020000.tar.gz # 文件备份
# config_20260425_020000.tar.gz  # 配置备份
# info_20260425_020000.txt       # 备份信息
```

### 3.2 查看备份

```bash
# 查看备份目录大小
du -sh backups/

# 查看备份文件列表
ls -lht backups/ | head -20

# 查看备份信息
cat backups/info_最新日期.txt
```

### 3.3 恢复数据

```bash
# 1. 停止服务
docker-compose down

# 2. 恢复数据库（假设备份文件为 db_20260425_020000.sql.gz）
gunzip -c backups/db_20260425_020000.sql.gz | docker exec -i drawwork-postgres psql -U postgres

# 3. 恢复文件
cd /opt/drawwork
tar -xzf backups/uploads_20260425_020000.tar.gz

# 4. 启动服务
docker-compose up -d

# 5. 验证
curl http://localhost/health
```

### 3.4 自动备份配置

```bash
# 编辑 crontab
crontab -e

# 添加每天凌晨2点自动备份
0 2 * * * /opt/drawwork/scripts/backup.sh >> /opt/drawwork/logs/backup.log 2>&1

# 查看定时任务
crontab -l

# 查看备份日志
tail -f /opt/drawwork/logs/backup.log
```

## 四、系统更新

### 4.1 应用更新

```bash
# 使用更新脚本
cd /opt/drawwork
./scripts/update.sh

# 或手动更新：
# 1. 备份
./scripts/backup.sh

# 2. 拉取最新代码
git pull

# 3. 构建并重启
docker-compose up -d --build
```

### 4.2 更新数据库

```bash
# 进入 API 容器执行迁移
docker-compose exec api npm run migrate

# 或手动执行 SQL
docker exec -i drawwork-postgres psql -U postgres -d drawwork < migrations/xxx.sql
```

### 4.3 回滚更新

```bash
# 1. 停止服务
docker-compose down

# 2. 恢复数据库到更新前
# 找到更新前的备份文件
gunzip -c backups/db_更新前日期.sql.gz | docker exec -i drawwork-postgres psql -U postgres

# 3. 恢复代码
git log --oneline  # 查看提交历史
git reset --hard HEAD~1  # 回滚到上一个版本

# 4. 重新构建并启动
docker-compose up -d --build
```

## 五、故障排查

### 5.1 服务无法启动

```bash
# 检查端口冲突
netstat -tlnp | grep -E ':(80|443|3000|3001|5432|6379|9000)'

# 检查磁盘空间
df -h

# 检查内存使用
free -h

# 查看详细错误日志
docker-compose logs --tail=200
```

### 5.2 数据库连接失败

```bash
# 检查数据库容器状态
docker-compose ps postgres

# 查看数据库日志
docker-compose logs postgres

# 测试连接
docker exec -it drawwork-postgres psql -U postgres -c "SELECT 1"

# 检查数据库配置
docker-compose exec api env | grep DATABASE
```

### 5.3 WebSocket 连接失败

```bash
# 检查 Yjs 服务状态
docker-compose ps yjs

# 查看 Yjs 日志
docker-compose logs yjs

# 测试 WebSocket 端口
curl -i -N -H "Connection: Upgrade" -H "Upgrade: websocket" \
  -H "Host: localhost" -H "Origin: http://localhost" \
  http://localhost:3001

# 检查 Nginx WebSocket 配置
docker exec drawwork-nginx nginx -t
```

### 5.4 文件上传失败

```bash
# 检查 Minio 状态
docker-compose ps minio

# 检查上传目录权限
ls -la data/uploads/

# 检查磁盘空间
df -h

# 查看 Minio 日志
docker-compose logs minio
```

### 5.5 性能问题

```bash
# 查看容器资源使用
docker stats --no-stream

# 查看数据库慢查询（需在 postgres 中启用）
docker exec drawwork-postgres psql -U postgres -c \
  "SELECT query, calls, mean_time FROM pg_stat_statements ORDER BY mean_time DESC LIMIT 10;"

# 查看 Redis 内存使用
docker exec drawwork-redis redis-cli info memory

# 重启服务释放内存
docker-compose restart
```

## 六、监控检查

### 6.1 健康检查

```bash
# API 健康检查
curl http://localhost/health

# 预期输出: healthy

# 完整健康检查（包括数据库等）
curl http://localhost/api/health/detailed
```

### 6.2 系统状态检查

```bash
# 创建检查脚本
cat > /opt/drawwork/scripts/check.sh << 'EOF'
#!/bin/bash
echo "=== DrawWork 系统状态检查 ==="
echo ""

echo "1. 容器状态:"
docker-compose ps

echo ""
echo "2. 磁盘使用:"
df -h /opt/drawwork

echo ""
echo "3. 内存使用:"
free -h

echo ""
echo "4. 服务健康检查:"
curl -s http://localhost/health || echo "❌ 服务异常"

echo ""
echo "5. 数据库连接:"
docker exec drawwork-postgres psql -U postgres -c "SELECT 1" > /dev/null 2>&1 && echo "✅ 数据库正常" || echo "❌ 数据库异常"

echo ""
echo "6. 最近错误日志:"
docker-compose logs --tail=20 2>&1 | grep -i error | tail -5

echo ""
echo "=== 检查完成 ==="
EOF

chmod +x /opt/drawwork/scripts/check.sh

# 执行检查
/opt/drawwork/scripts/check.sh
```

## 七、磁盘清理

### 7.1 清理日志

```bash
# 查看日志目录大小
du -sh logs/

# 清空日志文件（保留文件）
docker-compose exec api sh -c "truncate -s 0 /app/logs/*.log"
docker-compose exec nginx sh -c "truncate -s 0 /var/log/nginx/*.log"

# 或删除旧日志
find logs/ -name "*.log" -mtime +7 -delete
```

### 7.2 清理 Docker

```bash
# 查看 Docker 磁盘使用
docker system df

# 清理未使用的容器
docker container prune -f

# 清理未使用的镜像
docker image prune -f

# 清理未使用的卷（谨慎！）
docker volume prune -f

# 全面清理（除运行中的容器外）
docker system prune -f

# 全面清理包括卷（危险！）
docker system prune -a -f --volumes
```

### 7.3 清理旧备份

```bash
# 手动清理30天前的备份
find backups/ -name "*.gz" -mtime +30 -delete
find backups/ -name "*.txt" -mtime +30 -delete

# 查看备份目录大小
du -sh backups/
```

## 八、安全配置

### 8.1 修改默认密码

```bash
# 1. 修改数据库密码
docker-compose exec postgres psql -U postgres -c "ALTER USER postgres WITH PASSWORD 'new-strong-password';"

# 更新 .env 文件
vim config/.env
# 修改 DB_PASSWORD

# 重启服务
docker-compose restart api
```

### 8.2 配置 HTTPS

```bash
# 1. 准备 SSL 证书
mkdir -p /opt/drawwork/config/ssl
cp your-cert.pem /opt/drawwork/config/ssl/
cp your-key.pem /opt/drawwork/config/ssl/

# 2. 修改 nginx.conf，启用 HTTPS server 块
vim config/nginx.conf

# 3. 重启 Nginx
docker-compose restart nginx
```

### 8.3 访问控制

```bash
# 配置防火墙（仅允许内网访问）
ufw allow from 192.168.0.0/16 to any port 80
ufw allow from 10.0.0.0/8 to any port 80

# 拒绝外网访问
ufw deny 80
ufw enable
```

## 九、常见问题 FAQ

### Q1: 忘记管理员密码怎么办？

A: 使用数据库命令重置：
```bash
docker exec -it drawwork-postgres psql -U postgres -d drawwork
UPDATE users SET password_hash = '$2b$12$LQv3c1yqBWVHxkd0LHAkCOYz6TtxMQJqhN8/LewKyNiAYMyzJ/I2K' WHERE username = 'admin';
\q
# 密码重置为 admin123
```

### Q2: 如何导出某个画板的数据？

A: 使用管理后台或数据库查询：
```bash
docker exec drawwork-postgres psql -U postgres -d drawwork -c \
  "COPY (SELECT * FROM boards WHERE id = '画板ID') TO STDOUT;"
```

### Q3: 协作不同步怎么办？

A: 
1. 检查 Yjs 服务状态：`docker-compose ps yjs`
2. 重启 Yjs 服务：`docker-compose restart yjs`
3. 检查网络连接

### Q4: 上传文件大小限制怎么改？

A: 修改多个配置：
1. `docker-compose.yml` 中的 `client_max_body_size`
2. `.env` 中的 `UPLOAD_MAX_SIZE`
3. 重启服务：`docker-compose restart`

### Q5: 如何迁移到新服务器？

A:
1. 在原服务器执行备份：`./scripts/backup.sh`
2. 复制备份文件到新服务器
3. 在新服务器安装 Docker
4. 克隆项目代码
5. 恢复数据（见 3.3 恢复数据）
6. 启动服务
